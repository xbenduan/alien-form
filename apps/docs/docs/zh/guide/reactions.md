# 响应式联动 (Reactions)

联动是根据其他字段的值来派生当前字段状态的运行时机制。

## 心智模型

联动主要回答一个问题：

- 当某些依赖发生变化时，应该更新字段的哪个属性？

在 AlienForm 中，属性键名是显式的。Schema 并不是描述一个通用的副作用系统，而是描述对字段状态的赋值操作，例如更新 `display`（显示状态）、`pattern`（交互模式）、`title`（标题）、`props`（组件属性）或 `dataSource`（数据源）。

## 支持的规则类型

| 类型 | 用途 |
| --- | --- |
| `static` | 分配一个固定值 |
| `expression` | 根据表达式作用域计算出一个值 |
| `match` | 通过分支映射选择一个值 |
| `computed` | 委托给已注册的运行时处理函数 |

## 依赖解析

依赖项可以表示为：

- 数组，通过 `$deps[index]` 消费
- 对象，通过 `$deps.name` 消费

相对路径（如 `.source`）会相对于当前字段的父级路径进行解析。

## 为什么联动是可预测的

当前的实现并没有暴露任意的、命令式的联动 DSL。相反，它将联动限制在一组已知的目标键名中，并通过 `Form` 模型来统一执行。

这使得联动系统具有高度的可检查性，并允许核心层通过 `onError` 报告不支持的键名。
