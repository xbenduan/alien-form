# 数组字段

数组字段被建模为一个父字段加上一组子行字段。

## 什么时候会成为数组字段

当满足以下条件时，Schema 节点将被视为数组字段：

- `type` 为 `array`
- `items` 是一个对象 Schema
- `items.properties` 存在

在这种模式下，该字段会暴露数组辅助方法，并且 React 层会通过数组字段渲染器 `ArrayFieldRenderer` 来渲染它。

## 运行时行为

数组字段支持以下操作：

- `push(initialValues?)`：添加一行
- `remove(index)`：移除指定行
- `moveUp(index)`：上移一行
- `moveDown(index)`：下移一行
- `arrayItems`：获取所有行项

## 标识规则

移除某一行会原地重新索引后续的行字段，从而保留后续行的字段标识。这对于响应式 effect、校验状态以及行本地元数据都很重要。

## 渲染器契约

像 `ArrayCards` 和 `ArrayTable` 这样的数组组件会接收到以下 props：

- `rows`
- `onAdd`
- `onRemove`
- `onMoveUp`
- `onMoveDown`
- `disabled`
- `readOnly`
- `readPretty`

这使得数组 UI 组件成为核心数组模型之上的一层很薄的表现层。

## 适合的场景

- 动态增删行的商品规格、联系人、明细列表
- 每一行都需要成为独立字段集合的场景
- 需要保留行级状态、校验和局部联动的场景

## 不适合的场景

- 只是渲染一个简单字符串数组，但不需要行级字段状态
- 数据虽然是列表，但不需要作为字段树的一部分参与校验和联动

## 注意事项

- `arrayItems` 中的每一行都是真实字段实例集合，而不是普通数据对象
- 如果你的联动逻辑跨越多行或跨越数组与外层字段，通常应优先放到 `setup + form.effect(...)`
- 数组组件负责表现和交互，不应承担字段树重建或业务规则调度

## 一句话原则

数组字段的本质不是“渲染一个列表”，而是“让每一行都进入同一套字段运行时模型”。
