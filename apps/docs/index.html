Alien-Form 完整上手文档

这份文档以当前仓库里的 packages/core 和 packages/react 实现为准，目标不是只讲概念，而是让你看完后就知道：应该在哪写 schema、在哪写 handlers、在哪注册组件、页面里怎么渲染、x-* 怎么写、ctx 怎么用。

一句话理解：@alien-form/core 负责表单状态机和规则执行，@alien-form/react 负责把字段树渲染成 React 组件树。你真正需要写的业务代码，主要就 5 类：schema、handlers、components、decorators、页面入口。

1. 先建立正确心智模型

1
Schema 是唯一真相源

字段结构、标题、组件名、装饰器名、校验、联动，基本都写在 schema 里。

2
Core 不负责 UI

@alien-form/core 只关心字段树、值、规则、校验、提交，不认识 DOM。

3
React 只负责绑定

@alien-form/react 从 components 和 decorators 里按名字取组件来渲染。

4
复杂逻辑写 Handler

简单条件用 {{ ... }}，复杂逻辑、异步逻辑、跨字段编排写 @handler。

推荐上手方式： 先把一个最小表单跑起来，再逐步加入 x-reaction、x-effect、x-validate。不要一开始就把全部业务逻辑塞进 Schema。

2. 推荐目录结构：你应该在哪写什么代码

Alien-Form 不强制你的业务目录结构，但如果你希望“看一眼就知道代码放哪”，推荐按下面拆分：

src/
  alien-form/
    components.tsx      // 基础输入组件：Input、Select、ArrayCards、SectionCard...
    decorators.tsx      // 装饰器：FormItem
    handlers.ts         // @handler 对应的处理器实现
    schema.ts           // IFormSchema，字段结构和 x-* 规则都写这里
    form.ts             // createForm / useCreateForm 的统一封装（可选）
  pages/
    UserFormPage.tsx    // 页面入口，创建 form 并渲染 SchemaField
文件 你在这里写什么 为什么单独放
components.tsx 真正渲染输入控件的 React 组件，比如 Input、Select、ArrayCards UI 可复用，也便于替换设计系统
decorators.tsx 字段外层容器，比如 label、错误信息、描述文案的统一渲染 把“输入控件”和“表单项外壳”分离
handlers.ts 所有 @xxx 规则对应的函数实现 复杂逻辑集中管理，不污染 schema
schema.ts 字段定义、默认值、校验、联动、组件名、装饰器名 业务表单协议集中管理
UserFormPage.tsx 创建 form，注册 components / decorators，渲染 SchemaField，处理提交 页面层只做装配，不做细节逻辑
结论： x-* 的声明写在 schema.ts；@handler 的实现写在 handlers.ts；页面只做 form 创建和渲染。这样后期维护成本最低。

3. 从零搭一个可运行表单

3.1 先写基础输入组件：src/alien-form/components.tsx

Alien-Form 会把字段当前状态注入到你注册的组件里。对于 primitive 字段，最常见的 props 是：value、onChange、disabled、loading、dataSource，再加上 schema 里的 props。

import React from "react";

export const Input = ({ value, onChange, disabled, placeholder }) => {
  return (
    <input
      value={value ?? ""}
      disabled={disabled}
      placeholder={placeholder}
      onChange={(e) => onChange(e.target.value)}
    />
  );
};

export const Select = ({ value, onChange, disabled, dataSource = [] }) => {
  const stringValue = value === undefined || value === null ? "" : String(value);

  return (
    <select
      value={stringValue}
      disabled={disabled}
      onChange={(e) => {
        const selected = dataSource.find((item) => String(item.value) === e.target.value);
        onChange(selected ? selected.value : e.target.value);
      }}
    >
      <option value="">请选择</option>
      {dataSource.map((item) => (
        <option key={String(item.value)} value={String(item.value)}>
          {item.label}
        </option>
      ))}
    </select>
  );
};

export const NumberInput = ({ value, onChange, disabled, placeholder }) => {
  return (
    <input
      type="number"
      value={value ?? ""}
      disabled={disabled}
      placeholder={placeholder}
      onChange={(e) => onChange(e.target.value === "" ? undefined : Number(e.target.value))}
    />
  );
};

export const SectionCard = ({ title, description, children }) => {
  return (
    <section style={{ border: "1px solid #ddd", padding: 16, borderRadius: 8, marginBottom: 16 }}>
      {title ? <h4>{title}</h4> : null}
      {description ? <p style={{ color: "#666" }}>{description}</p> : null}
      {children}
    </section>
  );
};

export const ArrayCards = ({ rowNodes, rowFields, onAdd, onRemove, disabled }) => {
  return (
    <div style={{ display: "grid", gap: 12 }}>
      {rowNodes.map((row, index) => (
        <div key={row.id} style={{ border: "1px solid #ddd", padding: 12, borderRadius: 8 }}>
          <div>{rowFields[index].name}</div>
          <div>{rowFields[index].phone}</div>
          {!disabled ? (
            <button type="button" onClick={() => onRemove(index)}>
              删除这一行
            </button>
          ) : null}
        </div>
      ))}

      {!disabled ? (
        <button type="button" onClick={() => onAdd({})}>
          新增一行
        </button>
      ) : null}
    </div>
  );
};

export const formComponents = {
  Input,
  Select,
  NumberInput,
  SectionCard,
  ArrayCards,
};

3.2 再写装饰器：src/alien-form/decorators.tsx

装饰器负责字段标题、必填星号、错误信息、说明文案这些外层 UI。schema 中的 decorator 写什么名字，这里就要注册同名组件。

import React from "react";

export const FormItem = ({
  label,
  required,
  errors = [],
  warnings = [],
  description,
  children,
}) => {
  return (
    <div style={{ marginBottom: 16 }}>
      {label ? (
        <label style={{ display: "block", marginBottom: 8, fontWeight: 600 }}>
          {required ? <span style={{ color: "red" }}>*</span> : null}
          {label}
        </label>
      ) : null}

      <div>{children}</div>

      {description ? <div style={{ color: "#666", fontSize: 12 }}>{description}</div> : null}
      {errors[0] ? <div style={{ color: "red", fontSize: 12 }}>{errors[0].message}</div> : null}
      {!errors[0] && warnings[0] ? <div style={{ color: "#d97706", fontSize: 12 }}>{warnings[0].message}</div> : null}
    </div>
  );
};

export const formDecorators = {
  FormItem,
};

3.3 写 handlers：src/alien-form/handlers.ts

所有 @handler 都在这里实现。简单值计算可以直接 return，需要响应式监听时请 return ctx.effect(() => { ... })。

export const formHandlers = {
  calcTotal(ctx) {
    const price = Number(ctx.values.price || 0);
    const count = Number(ctx.values.count || 0);
    return price * count;
  },

  toggleInvoice(ctx) {
    return ctx.values.needInvoice ? "visible" : "none";
  },

  watchProvince(ctx) {
    return ctx.effect(() => {
      const province = ctx.values.province;
      const cityField = ctx.form.field("city");
      if (!cityField) return;

      if (!province) {
        cityField.setDataSource([]);
        ctx.set("city", "");
        return;
      }

      const list = getCitiesByProvince(province);
      cityField.setDataSource(list);
    });
  },

  checkConfirmPassword(ctx) {
    if (!ctx.value) return [];
    if (ctx.value !== ctx.values.password) {
      return [{ message: "两次输入的密码不一致", type: "error" }];
    }
    return [];
  },

  async checkUsernameUnique(ctx) {
    if (!ctx.value) return [];
    const exists = await fakeCheckUsername(ctx.value);
    return exists ? [{ message: "用户名已存在", type: "error" }] : [];
  },
};

function getCitiesByProvince(province) {
  if (province === "zhejiang") {
    return [
      { label: "杭州", value: "hangzhou" },
      { label: "宁波", value: "ningbo" },
    ];
  }

  if (province === "jiangsu") {
    return [
      { label: "南京", value: "nanjing" },
      { label: "苏州", value: "suzhou" },
    ];
  }

  return [];
}

async function fakeCheckUsername(username) {
  await new Promise((resolve) => setTimeout(resolve, 300));
  return username === "admin";
}

3.4 写 schema：src/alien-form/schema.ts

Schema 是整个表单的协议层。你在这里定义：字段类型、标题、组件、装饰器、静态校验、动态联动、副作用和动态校验。

export const userFormSchema = {
  type: "object",
  properties: {
    basicInfo: {
      type: "void",
      title: "基础信息",
      component: "SectionCard",
      properties: {
        username: {
          type: "string",
          title: "用户名",
          component: "Input",
          decorator: "FormItem",
          required: true,
          validate: {
            minLength: 3,
            maxLength: 20,
          },
          "x-validate": "@checkUsernameUnique",
        },
        password: {
          type: "string",
          title: "密码",
          component: "Input",
          decorator: "FormItem",
          required: true,
        },
        confirmPassword: {
          type: "string",
          title: "确认密码",
          component: "Input",
          decorator: "FormItem",
          "x-validate": "@checkConfirmPassword",
        },
      },
    },

    province: {
      type: "string",
      title: "省份",
      component: "Select",
      decorator: "FormItem",
      dataSource: [
        { label: "浙江", value: "zhejiang" },
        { label: "江苏", value: "jiangsu" },
      ],
      "x-effect": "@watchProvince",
    },

    city: {
      type: "string",
      title: "城市",
      component: "Select",
      decorator: "FormItem",
    },

    needInvoice: {
      type: "boolean",
      title: "是否开发票",
      component: "Select",
      decorator: "FormItem",
      dataSource: [
        { label: "是", value: true },
        { label: "否", value: false },
      ],
    },

    invoiceTitle: {
      type: "string",
      title: "发票抬头",
      component: "Input",
      decorator: "FormItem",
      "x-reaction": {
        display: "@toggleInvoice",
        required: "{{ $values.needInvoice === true }}",
      },
    },

    price: {
      type: "number",
      title: "单价",
      component: "NumberInput",
      decorator: "FormItem",
    },

    count: {
      type: "number",
      title: "数量",
      component: "NumberInput",
      decorator: "FormItem",
    },

    total: {
      type: "number",
      title: "总价",
      component: "NumberInput",
      decorator: "FormItem",
      disabled: true,
      "x-reaction": {
        value: "@calcTotal",
      },
    },

    contacts: {
      type: "array",
      title: "联系人列表",
      component: "ArrayCards",
      decorator: "FormItem",
      items: {
        type: "object",
        properties: {
          name: {
            type: "string",
            title: "联系人姓名",
            component: "Input",
            decorator: "FormItem",
          },
          phone: {
            type: "string",
            title: "联系人电话",
            component: "Input",
            decorator: "FormItem",
            validate: {
              format: "phone",
            },
          },
        },
      },
    },
  },
};

3.5 页面入口：src/pages/UserFormPage.tsx

页面里只负责三件事：创建 form、注入 UI 注册表、处理提交。

import React from "react";
import {
  useCreateForm,
  FormProvider,
  SchemaField,
  useFormSubmit,
} from "@alien-form/react";

import { userFormSchema } from "../alien-form/schema";
import { formHandlers } from "../alien-form/handlers";
import { formComponents } from "../alien-form/components";
import { formDecorators } from "../alien-form/decorators";

function SubmitButton() {
  const { submit, submitting } = useFormSubmit();

  return (
    <button
      type="button"
      disabled={submitting}
      onClick={() =>
        submit(async (values) => {
          console.log("提交值", values);
          return values;
        })
      }
    >
      {submitting ? "提交中..." : "提交"}
    </button>
  );
}

export default function UserFormPage() {
  const form = useCreateForm({
    schema: userFormSchema,
    handlers: formHandlers,
    initialValues: {
      needInvoice: false,
      contacts: [{}],
    },
    scope: {
      appName: "alien-form-demo",
    },
    onError(error) {
      console.warn("[alien-form runtime error]", error);
    },
  });

  return (
    <FormProvider
      form={form}
      components={formComponents}
      decorators={formDecorators}
    >
      <SchemaField />
      <SubmitButton />
    </FormProvider>
  );
}
至此你应该已经清楚：
  • schema.ts 写表单协议。
  • handlers.ts 写业务逻辑。
  • components.tsx 写输入组件。
  • decorators.tsx 写表单项外壳。
  • UserFormPage.tsx 负责把这些拼起来。

4. Schema 全量说明

根 schema 必须是 type: "object"。字段定义都放在 properties 里。常用字段属性如下：

属性 作用 你一般在什么时候写
type 字段类型，常见为 string、number、boolean、object、array、void 定义字段结构时一定会写
title 字段标题，通常交给 decorator 展示 几乎所有可见字段都建议写
description 字段说明文案 需要解释输入规则时
default 字段默认值 希望 reset 回到某个默认状态时
required 是否必填，也可由父 object 的 required: [] 指定 静态必填
display 显示状态：visible / hidden / none 静态控制显隐
disabled 是否禁用 静态禁用字段
validate 静态校验规则 长度、格式、最小值、正则等
component 要渲染的组件名 告诉 React 侧用哪个输入组件
props 透传给组件的 props 例如 placeholder、size 等
decorator 要渲染的装饰器名 需要统一 label / errors 外壳时
decoratorProps 透传给装饰器的 props 控制布局、提示样式等
dataSource 选项数据源 Select、Radio、CheckboxGroup 一类组件
x-reaction 当前字段的派生属性逻辑 显示隐藏、禁用、标题切换、赋值
x-effect 副作用逻辑 监听字段变化后请求接口、重置其他字段
x-validate 动态校验逻辑 跨字段校验、异步接口校验

4.1 validate 支持哪些静态校验

规则 含义
required必填
minimum / maximum数字最小值 / 最大值
exclusiveMinimum / exclusiveMaximum严格大于 / 严格小于
multipleOf数字必须是某个值的倍数
minLength / maxLength字符串长度限制
pattern正则匹配
format内置格式：email、url、phone、number、integer、idcard、ip、zip
minItems / maxItems数组项数量限制
uniqueItems数组元素必须唯一
const值必须等于某个常量
message统一错误文案

4.2 表达式里可以用什么

Alien-Form 支持 {{ ... }} 表达式，但它不是完整 JavaScript，而是一个安全子集。你可以使用：

  • 布尔运算、比较运算、算术运算、三元表达式
  • 对象字面量、数组字面量、成员访问
  • $values、$form、$self、$row、$path、$get()、$project()
  • FormConfig.scope 注入的变量
不要在表达式里写： 函数调用、赋值、模板字符串、window、globalThis、eval、prototype、constructor。这些都不允许。

5. x-* 全量说明

5.1 x-reaction：给“当前字段”算派生属性

x-reaction 的本质是：根据上下文，为当前字段的某个属性算出新值。常见目标如下：

target key 作用 适用字段
value修改当前字段值仅 primitive 字段
rows替换数组行仅 array 字段
display控制显示 / 隐藏 / 不参与提交所有字段
disabled控制禁用所有字段
required控制是否必填所有字段
title动态标题所有字段
description动态说明所有字段
props动态组件 props所有字段
decoratorProps动态装饰器 props所有字段
component切换组件名，或 [componentName, props]所有字段
decorator切换装饰器名，或 [decoratorName, props]所有字段
dataSource动态选项所有字段
 
"x-reaction": {
  display: "{{ $values.needInvoice ? 'visible' : 'none' }}",
  required: "{{ $values.needInvoice === true }}",
  props: "{{ { placeholder: $values.needInvoice ? '请输入发票抬头' : '当前无需填写' } }}"
}
hidden 和 none 的区别非常重要：
  • hidden：只是隐藏 DOM，字段值仍然存在，也会参与提交和校验。
  • none：该字段的值在投影时会被忽略，不参与 values / submit / validate 聚合。

5.2 x-effect：做副作用，不是给当前字段属性赋值

适合写这些事：

  • 监听某字段变化后，请求接口
  • 重置其他字段
  • 动态设置别的字段的 dataSource
  • 安装一个长期响应式监听
"x-effect": "@watchProvince"
watchProvince(ctx) {
  return ctx.effect(() => {
    const province = ctx.values.province;
    const cityField = ctx.form.field("city");
    if (!cityField) return;

    if (!province) {
      cityField.setDataSource([]);
      ctx.set("city", "");
      return;
    }

    cityField.setDataSource(fetchCityOptions(province));
  });
}

推荐做法： 只要是“持续监听”的逻辑，就让 handler return ctx.effect(...)，这样表单销毁时可以自动清理。

5.3 x-validate：写动态校验

x-validate 会在静态校验之后执行，适合：

  • 密码确认
  • 开始日期不能大于结束日期
  • 用户名是否重名
  • 不同角色对应不同校验规则

返回值支持这些形式：

返回值 含义
undefined / null / true校验通过
false失败，默认错误文案为 Invalid value
"错误文案"失败，使用该字符串作为错误信息
{ message: "错误文案", type: "error" }失败，返回单个错误对象
[...]返回多个错误对象
Promise<...>支持异步校验
 
"x-validate": "@checkConfirmPassword"
checkConfirmPassword(ctx) {
  if (!ctx.value) return [];
  if (ctx.value !== ctx.values.password) {
    return [{ message: "两次输入的密码不一致", type: "error" }];
  }
  return [];
}

6. Handlers 与 ctx 全量说明

Schema 里写 "@handlerName"，运行时会从 createForm({ handlers }) 里找到同名函数执行。handler 会收到两个参数：

(ctx, form) => { ... }

实际使用时，通常只用第一个参数 ctx 就够了，因为 ctx.form 已经能拿到表单实例。

6.1 ctx 有哪些字段

ctx 字段 说明 典型用途
ctx.field 当前规则所属字段节点 判断当前字段、直接调用字段实例方法
ctx.form 表单实例 查别的字段、提交、validate、读取表单聚合值
ctx.path 当前字段路径 调试、日志、条件分流
ctx.key 当前规则作用的 key，比如 display、value 写通用 handler 时区分目标
ctx.kind 当前规则类型：x-reaction / x-effect / x-format / x-validate 写通用处理器或调试
ctx.schema 当前字段 schema 按 schema 配置做分支
ctx.row 当前字段所在的数组行 数组子项联动
ctx.scope createForm({ scope }) 注入的自定义作用域 表达式和 handler 共享的外部常量
ctx.values 当前表单聚合值 最常用，跨字段读值
ctx.value 当前值，常见于 x-validate 校验当前字段输入
ctx.get(selector) 按 selector 读取值 读取兄弟字段、数组行字段、绝对路径字段
ctx.set(selector, value) 按 selector 设置值 给其他 primitive 字段赋值
ctx.project(selector?) 获取某个字段的投影结果 读取 object / array / void 展开后的数据
ctx.effect(runner) 安装响应式副作用 监听依赖变化后自动执行逻辑

6.2 在 Handlers 中如何使用响应式数据

如果你只是做一次性计算，比如给当前字段算一个值，那么直接返回结果即可：

calcTotal(ctx) {
  return Number(ctx.values.price || 0) * Number(ctx.values.count || 0);
}

如果你要做“当某些值变化时，自动重新执行”的逻辑，就必须使用 ctx.effect：

watchPrice(ctx) {
  return ctx.effect(() => {
    const price = ctx.values.price;
    const count = ctx.values.count;
    ctx.set("total", Number(price || 0) * Number(count || 0));
  });
}

这里的关键点是：

  • 在 ctx.effect 回调里读取到的响应式数据，会被自动追踪。
  • 例如 ctx.values.price、ctx.values.count 变化时，回调会自动重新执行。
  • 你应当把 ctx.effect(...) 返回出去，让框架接管清理。
经验法则：
  • “算一个值”用 x-reaction + 直接返回。
  • “监听变化后做副作用”用 x-effect + return ctx.effect(...)。
  • “检查是否合法”用 x-validate。

6.3 ctx.get() 的 selector 怎么写

selector 说明 示例
$value 当前字段值 ctx.get("$value")
$path 当前字段路径 ctx.get("$path")
$row.name 当前数组行中的兄弟字段 ctx.get("$row.name")
contacts[].name 取数组每一行某个字段的值列表 ctx.get("contacts[].name")
./sibling 相对当前字段所在层级读取兄弟字段 ctx.get("./confirmPassword")
absolute.path 绝对路径读取 ctx.get("invoiceTitle")

6.4 ctx.set() 能做什么，不能做什么

  • 能做： 给 primitive 字段赋值，比如字符串、数字、布尔值。
  • 适合： ctx.set("total", 100)、ctx.set("city", "")。
  • 不能做： 直接给 object / array 字段整体赋值。
  • 数组整体更新： 应通过数组字段实例的 setRows() 或 push/remove/move。

7. 组件和装饰器分别会收到什么 props

7.1 primitive 组件会收到什么

如果字段是基础类型，Alien-Form 会给组件注入：

  • value
  • onChange
  • disabled
  • loading
  • dataSource（如果存在）
  • schema 的 props
const Input = ({ value, onChange, disabled, loading, placeholder }) => {
  return (
    <input
      value={value ?? ""}
      disabled={disabled || loading}
      placeholder={placeholder}
      onChange={(e) => onChange(e.target.value)}
    />
  );
};

7.2 decorator 会收到什么

  • label
  • required
  • errors
  • warnings
  • description
  • validateStatus
  • schema 的 decoratorProps
  • children

7.3 array 组件会收到什么

数组组件不是简单的输入框，它会拿到一整套行控制能力：

  • field：数组字段节点
  • rows：每一行预渲染后的 ReactNode 列表
  • rowNodes：每一行的 RowNode
  • rowFields：每一行按字段名组织的 ReactNode 映射
  • onAdd / onRemove / onMoveUp / onMoveDown / onMove
  • disabled

因此数组组件最适合放“卡片列表”、“表格行编辑”、“可新增删除的表单块”。

7.4 object 与 void 组件怎么理解

  • object：保留结构，适合真正的数据对象。
  • void：只做布局分组，不单独产出值，子字段最终会拍平合并回父级。

8. 数组、对象、void 这三类字段怎么选

类型 什么时候用 提交结果长什么样
object 你希望数据保持嵌套结构 保留对象层级
array 一组可增删改移动的行数据 提交为数组
void 只是为了布局、分组、分栏、分区 自身不单独出值，子字段拍平并入父级
尤其注意： void 很适合拿来做“基础信息”“配送信息”“发票信息”这种区块，但它不是一个真正的数据容器。

9. 表单实例 API 与提交流程

你平时最常用的表单实例方法如下：

方法 作用 示例
form.field(path)拿字段节点form.field("city")
form.get(selector)按 selector 读值form.get("contacts[].name")
form.set(selector, value)按 selector 设值form.set("city", "hangzhou")
form.project(selector?)获取字段投影结果form.project("contacts")
form.setValues(values)批量设值form.setValues({ city: "hangzhou" })
form.reset()重置form.reset()
form.validate()手动校验await form.validate()
form.submit(onSubmit?)校验通过后提交await form.submit(console.log)
form.onError(listener)监听非致命运行时错误form.onError(console.warn)

9.1 提交流程发生了什么

  1. 调用 form.submit()
  2. 框架先跑 validate()
  3. 如果有错误，抛出 Validation failed
  4. 如果通过，返回当前投影后的 form.values()
  5. 如果传了 onSubmit，则把 values 传进去执行
try {
  const result = await form.submit(async (values) => {
    await api.save(values);
    return values;
  });
  console.log("提交成功", result);
} catch (error) {
  console.log("提交失败", error);
}

10. React Hooks 怎么用

如果你不想一路 props 透传，可以直接在自定义组件里读表单状态：

Hook 作用
useForm()拿当前表单实例
useFieldAtoms(path)拿字段节点
useFieldValue(path)拿字段值
useFieldErrors(path)拿字段错误
useFieldDisplay(path)拿显示状态

import { useFieldValue, useFormValues } from "@alien-form/react";

function DebugPanel() {
  const city = useFieldValue("city");
  const values = useFormValues();

  return (
    <pre>
      当前城市：{JSON.stringify(city, null, 2)}
      全量表单值：{JSON.stringify(values, null, 2)}
    </pre>
  );
}

11. 最容易踩坑的地方

坑点 正确理解
以为 SchemaField schema={...} 能切换 schema 当前实现实际渲染的是 form.schema，不是传入的 props，所以一般通过重建 form 来切换 schema。
把复杂副作用写进表达式 表达式只适合简单派生，复杂逻辑用 @handler。
把跨字段更新都写成 x-reaction x-reaction 更适合给“当前字段”算属性；跨字段编排更适合 x-effect 或直接字段实例操作。
忘记注册 component / decorator schema 里的名字必须能在 components / decorators 映射表里找到。
误以为 hidden 不会提交 hidden 只是隐藏，值仍保留；真正不参与聚合的是 none。
用 ctx.set() 给 array / object 整体赋值 ctx.set() 只适合 primitive 字段。数组请操作数组字段实例。
把 void 当成普通对象容器 void 只做布局，子字段最终会并入父级。
以为 x-format 已经可用 当前代码里只有类型声明，没有实际执行逻辑，当前不要依赖它。
以为 validateFirst 已经生效 当前实现里这个配置项未实际使用。

12. 最后一张速查表

  • 写字段结构： 去 schema.ts
  • 写组件： 去 components.tsx
  • 写表单项外壳： 去 decorators.tsx
  • 写复杂联动 / 副作用 / 异步逻辑： 去 handlers.ts
  • 把它们拼起来： 去页面文件里 useCreateForm + FormProvider + SchemaField
  • 简单显隐禁用： 优先写 {{ ... }}
  • 复杂逻辑： 优先写 @handler
  • 跨字段副作用： 用 x-effect + return ctx.effect(...)
  • 动态校验： 用 x-validate
  • 布局分组： 用 void
  • 真正嵌套数据： 用 object
  • 行列表： 用 array
